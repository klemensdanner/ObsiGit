# Rechenwerk
- rechnet (wow)
- Bitshiftoperationen (eigentlich einfach durch 2 oder mal 2)
	- kann das Rechenwerk direkt oder sogar direkt das Register
		- geht mittels "schiefen" Leitungen vom einen in ein anderes Register, wo einfach die Bits um 1 nach rechts oder links verschoben werden
- Vergleichsoperationen
- bitweise logische Operationen
- Werte "durchschleusen"
- Konstanten generieren als Startwert für logische Funktionen
- Operation wird als Parameter übergeben f(x,y,operation) -- solche unterstützte Operationen sind direkt in der Hardware realisiert
Eine solche Einheit im Rechenwert nennt man "Arithmetisch-Logische Einheit" (Arithmetic Logic Unit, ALU)
- X und Y sind Register: die inputs werden gespeichert, da die Berechnung Zeit braucht
- Wortbreite: Anzahl der Bits die eine ALU pro Operation verarbeiten kann: heutzutage normalerweise 64 Bits
- Flags Register: boolean Werte - einfache Informationen werden mitgeliefert (Liste von Bits für Zusatzinformationen), z.B.
	- Eigenschaften vom Ergebnis (Status Flags)
		- Zero Flag (ZF): Ergebnis war null
		- Sign Flag (SF): Ergebnis null
		- Parity Flag (PF): gerade Parität
	- Informationen über letzte Operation
		- z.B. Overflow etc. mehr in PDF
	- Steuer Flags (vom Programmmierer gesetzt)

### Carry-Look-Ahead Adder
[[4-Rechenwerk.pdf#page=10]]

- Berechnet alle Carries gleichzeitig anhand einer Schaltfunktion
	- propagiert: falls ein carry reinkommt, geht auch eins raus z.B. bei 0 + 1
	- generiert: hier kommt fix ein carry out z.B. bei 1 + 1 --egal welches carry in

- Allgemein: mit zusätzlichen Gattern lassen sich schnelle Schaltungen bauen
